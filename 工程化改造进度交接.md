## 2. 已完成的关键改动

### 2.1 fallback_agent

新增文件：

```text
cloud_agent/agent/agents/fallback_agent.py
```

作用：

- 处理非云平台问题
- 处理系统能力范围外问题
- 避免未知问题默认进入 `product_agent`
- 回复中说明当前系统支持范围

典型路由：

```text
今天天气怎么样？ -> fallback_agent
帮我写一首关于云服务器的诗 -> fallback_agent
```

### 2.2 support_agent

新增文件：

```text
cloud_agent/agent/agents/support_agent.py
```

作用：

- 处理 ECS SSH 失败
- 处理安全组端口不通
- 处理公网 IP 无法访问
- 处理 CPU / 内存异常
- 处理 RDS 连接失败
- 处理实例启动失败或状态异常

当前实现：

- 本地规则版排查清单
- 暂不调用 MCP 工具
- 后续可以接入实例状态、安全组、网络 ACL、监控指标等工具

典型路由：

```text
我的 ECS 无法 SSH 连接 -> support_agent
安全组 22 端口不通怎么办？ -> support_agent
RDS 连接失败，ECS 连不上数据库 -> support_agent
```

### 2.8 MCP 工具 timeout / retry

修改文件：

```text
cloud_agent/agent/agents/billing_agent.py
cloud_agent/agent/core/workflow/tool_audit.py
```

实现位置：

```python
UserIdInjector
```

能力：

- 使用 `asyncio.wait_for()` 包裹 MCP 工具调用
- 默认 timeout：30 秒
- 默认 retry：0 次
- 支持环境变量：
  - `CLOUD_AGENT_TOOL_TIMEOUT_SECONDS`
  - `CLOUD_AGENT_TOOL_RETRY_COUNT`
- 支持 runtime config：
  - `tool_timeout_seconds`
  - `tool_retry_count`
- 只重试：
  - `TimeoutError`
  - `ConnectionError`
  - `OSError`
- 不重试：
  - `ValueError` 等参数类错误

扩展审计字段：

```json
{
  "attempt": 1,
  "max_attempts": 2,
  "timeout_seconds": 1.0,
  "retryable": true
}
```

### 2.14 deep_research 反思循环防护

```text

```

新增状态字段：

```python

```

主要变化：

- `reflect_node` 对补搜词去重
- 去重来源包括：
  - 原始 `search_plan`
  - 已有 `supplementary_queries`
  - `web_search_trace`
  - `local_rag_trace`
- 如果没有新补搜词，设置：

```text

```

- 补搜轮如果 `evidence_pool` 没有增长，`no_new_evidence_rounds + 1`
- 图路由遇到以下情况直接进入 `write`：
  - `reflection_stop_reason` 非空
  - `no_new_evidence_rounds >= 1`
  - `iteration >= max_iterations`

    - `X-Authenticated-Tenant-Id`

    - `CLOUD_AGENT_AUTH_TENANT_HEADER`

  - 401 响应固定为 `{"detail":"authentication_required"}`，不包含明文 user_id、tenant_id、query、prompt、completion、matched_question 或异常 message。

- `chat.py` 只在 Web 边界调用 `resolve_authenticated_identity_from_request()`，随后把 `authenticated_user_id` / `authenticated_tenant_id` 传入既有 `stream_chat()`。
